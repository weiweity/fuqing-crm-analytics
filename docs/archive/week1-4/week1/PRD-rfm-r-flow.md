## 1. 产品定位

R 区间流转看板是 **RFM 分析体系的第 3 个 Tab**，聚焦**老客在不同休眠深度下的回购唤醒表现**。它与「流转矩阵」「桑基图」并列，共同构成完整的 RFM 运营洞察闭环。

| Tab | 核心问题 |
|-----|---------|
| 流转矩阵 | 人群在象限之间怎么流？ |
| 桑基图 | 流转路径的可视化呈现 |
| **R 区间流转** | **不同休眠时长的老客，各自被唤醒的效率如何？** |

---

## 2. 为什么只看老客、排除 cutoff 前首次购买的新客？

**核心原因：R 区间是基于「距 cutoff 的最近一次购买天数」构建的，它天然要求用户在 cutoff 之前至少有过 1 次购买记录。**

- **新客** = 在 cutoff 之后首次购买的用户，他们在 cutoff 之前没有购买记录，无法计算 R 值，也不属于任何休眠区间。
- **R 区间流转的业务目标** = 诊断「已购客池子」的唤醒效率，回答：近 1 个月的老客复购率多少？沉睡 1 年的客户还能唤醒多少？
- 如果把新客混进来，分母会被「没有历史的人群」稀释，回购率失去可比性，运营策略也会错位（新客是做首购转化，老客是做唤醒复购）。

因此，这张表的统计口径严格限定为：**cutoff 之前至少有过 1 次有效购买的用户**。

---

## 3. R 区间定义

基于 Excel 设计并扩展「近两年」区间：

| 区间名称 | 距 cutoff 的最近一次购买天数 | 说明 |
|---------|---------------------------|------|
| 近1个月已购客 | 0 - 30 天 | 活跃老客，重点维护 |
| 近2-3个月已购客 | 31 - 90 天 | 轻度沉睡，温和唤醒 |
| 近4-6月已购客 | 91 - 180 天 | 中度沉睡，优惠券刺激 |
| 近7-12个月已购客 | 181 - 365 天 | 深度沉睡，大促触达 |
| 近两年已购客 | 366 - 730 天 | 新增：超长周期沉睡，评估长期唤醒价值 |
| 2年外已购客 | > 730 天 | 超沉睡客户，低成本维护 |
| 已购客 TTL | 全部老客（R ≥ 0） | 汇总行 |

## 4. 指标口径

| 指标 | 定义 | 计算方式 | 展示格式 |
|------|------|---------|---------|
| **历史人群量级** | cutoff 前最后一次购买落在该 R 区间的用户数 | `COUNT(DISTINCT user_id)` | 整数，千分位 |
| **回购人数** | 上述历史人群中，在当前分析期内再次购买的用户数 | `COUNT(DISTINCT 回购 user_id)` | 整数，千分位 |
| **回购率** | 回购人数 / 历史人群量级 | 除法 | 百分比，1位小数 |
| **回购金额** | 回购用户在当前分析期内的 GSV 总和 | `SUM(actual_amount)` | ¥X.X 万 |
| **回购金额占比** | 该区间回购金额 / 所有区间回购金额合计 | 除法 | 百分比，1位小数 |

**数据过滤**：遵循语义层统一规则 `is_goujinjin = FALSE AND is_refund = FALSE AND order_status != '交易关闭'`。

## 5. 表格结构设计

严格复用人群看板「30 指标对比」的三栏年份布局 + YOY 列设计：

```
R 区间      | 当前年 (5列)                | YOY (5列)                | 前年 (5列)
-----------|----------------------------|--------------------------|---------------------------
           | 历史人群 | 回购人数 | 回购率 | 回购金额 | 占比 | ...同比... | ...前年绝对值...
```

- **第一列固定**：`R 区间`，左对齐，固定列宽
- **当前年**：5 个子列，展示原始绝对值
- **YOY**：5 个子列，展示相对去年的增长率
- **前年**：5 个子列，展示去年的原始绝对值
- **最后一行 TTL**：汇总行，字体加粗

## 6. 可视化设计：回购率 3 年对比柱状图

在表格上方放置一个分组柱状图，帮助快速识别各休眠层级的唤醒效率变化。

| 元素 | 设计 |
|------|------|
| **X 轴** | 6 个 R 区间（不含 TTL） |
| **Y 轴** | 回购率（%） |
| **系列** | 当前年（#2563eb 实心柱）、去年（#60a5fa 实心柱）、前年（#94a3b8 实心柱） |
| **Tooltip** | 显示精确回购率，保留 2 位小数 |
| **交互** | hover 高亮当前柱子，其他柱子降低透明度 |

**图表价值**：
- 一眼看出「近 1 个月」和「近 2-3 个月」的回购率是否在下滑
- 判断「近两年」超长周期客户的唤醒效率是否值得投入

## 7. 筛选器联动规则

完全复用人群看板的筛选器体系：

| 筛选器 | 联动效果 |
|--------|---------|
| **日期范围 / 周期** | 决定当前分析期 `start_date ~ end_date`，并自动计算同比期和前年期 |
| **渠道** | 历史人群和当期订单均按渠道过滤；选具体渠道时，只看该渠道的历史老客在该渠道的回购 |
| **cutoff** | 统一由 `PeriodBuilder` 或自定义日期计算：`cutoff = start_date - 1 天` |

**数据刷新机制**：`filterStore` 任一字段变化 → `queryParams` 变化 → `useQuery` 自动重新请求 → 图表 + 表格同步刷新。
