## 二、过滤条件层（filters.py）

### 2.1 基础过滤条件 / Base Filters

| 方法 | 中文名称 | SQL 片段 | 说明 |
|------|---------|---------|------|
| `OrderFilters.valid_order()` | 有效订单 | `order_status LIKE '%成功%' AND is_goujinjin = FALSE AND is_refund = FALSE` | **GSV口径基础**，剔除退款和购物金 |
| `OrderFilters.order_status_ok()` | 订单状态成功 | `order_status LIKE '%成功%'` | 基础状态过滤 |
| `OrderFilters.not_goujinjin()` | 非购物金 | `is_goujinjin = FALSE` | 剔除购物金订单 |
| `OrderFilters.not_refund()` | 非退款 | `is_refund = FALSE` | 剔除退款订单 |
| `OrderFilters.is_member()` | 会员订单 | `is_member = TRUE` | 会员筛选 |

### 2.2 时间过滤 / Time Filters

| 方法 | 中文名称 | SQL 片段 | 说明 |
|------|---------|---------|------|
| `OrderFilters.pay_time_between(start, end)` | 付款时间范围（闭区间） | `pay_time >= ? AND pay_time <= ?` | start/end 格式 `YYYY-MM-DD HH:MM:SS` |
| `OrderFilters.pay_time_between_dates(start, end)` | 付款日期范围（闭区间） | `pay_time >= ? AND pay_time <= ?` | start/end 格式 `YYYY-MM-DD`，自动补全时间 |

### 2.3 渠道过滤 / Channel Filters

| 方法 | 中文名称 | SQL 片段 | 说明 |
|------|---------|---------|------|
| `OrderFilters.channel_in(channels)` | 渠道 IN 列表 | `channel IN (?, ?, ...)` | 空列表返回 `1=1` |
| `OrderFilters.dimension_eq(dim, value)` | 维度精确匹配 | `COALESCE(dim, '未知') = ?` | 带空值保护 |

### 2.4 组合器 / Builders

| 类 | 中文名称 | 核心方法 | 说明 |
|----|---------|---------|------|
| `FilterBuilder` | 动态过滤构造器 | `with_metric_type()` / `with_time_range()` / `with_channels()` / `with_member_only()` / `build()` | 链式调用生成完整 WHERE 子句 |
| `AmountExprBuilder` | 金额表达式构造器 | `gsv(column)` / `sum_gsv(column)` | GSV 专用 CASE WHEN |

### 3.2 GMV vs GSV 核心区别 / GMV vs GSV（v2.0 更新）

| 指标 | 中文名 | 计算方式 | 包含退款 | 包含购物金 |
|------|--------|---------|---------|---------|
| **GMV** | 商品交易总额 | `SUM(CASE WHEN is_goujinjin=FALSE THEN actual_amount ELSE 0 END)` | ✅ 是 | ❌ 否（v2.0变更） |
| **GSV** | 有效销售额 | `SUM(CASE WHEN is_goujinjin=FALSE AND is_refund=FALSE THEN actual_amount ELSE 0 END)` | ❌ 否 | ❌ 否 |

> ⚠️ **v2.0变更**：GMV原含购物金，现与GSV一样剔除购物金。日常分析默认使用GSV；GMV仅用于对比退款场景。

## 四、新老客口径（segments.py / time.py）

### 4.1 新老客判定逻辑 / New vs Old Customer Logic

```
时间窗口：T1 ~ T2（T1=开始日期，T2=结束日期）
cutoff = T1 - 1天

用户判定：
  新客 = 窗口内有购买 AND 首购日期 > cutoff
  老客 = 窗口内有购买 AND 首购日期 ≤ cutoff
```

| 字段 | 中文名 | 说明 |
|------|--------|------|
| `is_new` | 新客标记 | `user_first_purchase.first_pay_date > cutoff` |
| `is_old` | 老客标记 | `user_first_purchase.first_pay_date ≤ cutoff` |

### 4.2 RFM 阈值 / RFM Thresholds

| 维度 | 中文名 | 阈值数组 | 说明 |
|------|--------|---------|------|
| R（最近） | 最近购买天数 | `[14, 30, 60, 90]` | R=5 → 最近14天内；R=1 → 90天以上 |
| F（频次） | 有效订单数 | `[1, 2, 3, 5]` | F=5 → 6次以上；F=1 → 1次 |
| M（金额） | 累计消费 | `[100, 300, 500, 1000]` | M=5 → 1000元以上；M=1 → 100元以下 |

### 5.2 RFM 评分计算 / RFM Score Calculation

```sql
-- R评分：天数越少分越高
R = CASE
    WHEN recency_days < 14 THEN 5
    WHEN recency_days < 30 THEN 4
    WHEN recency_days < 60 THEN 3
    WHEN recency_days < 90 THEN 2
    ELSE 1
END

-- F评分：频次越高分越高
F = CASE
    WHEN frequency >= 6 THEN 5
    WHEN frequency >= 4 THEN 4
    WHEN frequency = 3 THEN 3
    WHEN frequency = 2 THEN 2
    ELSE 1
END

-- M评分：金额越高分越高
M = CASE
    WHEN monetary >= 1000 THEN 5
    WHEN monetary >= 500 THEN 4
    WHEN monetary >= 300 THEN 3
    WHEN monetary >= 100 THEN 2
    ELSE 1
END
```

## 七、时间周期（time.py）（v2.0 新增 WTD/YTD）

### 7.1 周期模式 / Period Modes（v2.0 新增 WTD/YTD）

| 模式 | 中文名 | 当期范围 | 默认 |
|------|--------|---------|------|
| **`wtd`** | **当周至今（默认）** | 当周周一 → 昨天（t-1） | ✅ 前端默认 |
| `mtd` | 当月至今 | 当月1日 → 昨天（t-1） | — |
| `ytd` | 当年至今 | 1月1日 → 昨天（t-1） | — |
| `lookback` | 回溯期 | end=指定日期，start=end-N天 | — |
| `mom` | 环比 | 前一个等长周期 | — |
| `yoy` | 同比 | 去年同月同期 | — |
| `free` | 自由时间段 | 用户选两个日期，自行计算同期 | — |

### 7.2 WTD 详细说明

> **WTD 为前端默认模式**，截止昨天（t-1），不含当天。

```
today = 2026-04-16
yesterday = 2026-04-15

WTD 当期：     2026-04-13（周一）~ 2026-04-15
WTD 环比上周： 2026-04-06（周一）~ 2026-04-12
WTD 同比去年： 2025-04-13 ~ 2025-04-12（同期天数一致）
```

### 7.3 所有周期截止规则

> **所有固定周期（MTD/WTD/YTD）均不含当天**，截止到昨天（t-1）。

### 7.4 DateRange 结构

```python
@dataclass
class DateRange:
    start: str   # YYYY-MM-DD，窗口开始
    end: str     # YYYY-MM-DD，窗口结束（当天前一秒）
    cutoff: str  # YYYY-MM-DD，新老客判定边界 = start - 1天
```

assert goujinjin_rate < 0.30,  f"购物金率 {goujinjin_rate:.1%} 异常（>30%），ETL 中止"
