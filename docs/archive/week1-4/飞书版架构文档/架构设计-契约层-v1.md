## 1. 定位与核心原则

契约层（Contract Layer）是**前后端数据交互的唯一真实来源**，位于 FastAPI（main.py）与前端（Vue3）之间。

### 核心原则

> ✅ **单一真实来源**：所有 Pydantic Request/Response 模型定义在 `backend/contracts/schemas.py`，禁止在其他位置定义。

> ✅ **禁止内联**：main.py 中禁止内联定义 Pydantic 模型，全部从 contracts 导入。

> ✅ **类型驱动开发**：字段变更 → 修改 schemas.py → 重新生成 TypeScript 类型 → 前端自动获得类型提示。

> ✅ **变更必通知**：修改 schemas.py 后必须执行 `npm run gen:types`，前端才能同步。

### 数据流向

```
前端 Vue3 (TypeScript)
         ↓ 类型推断（自动生成）
OpenAPI JSON (/openapi.json)
         ↓ openapi-typescript 自动生成
TypeScript 类型（src/api/types.ts）
         ↑
后端 FastAPI
         ↓ Pydantic 模型
contracts/schemas.py  ← 契约层唯一真实来源
         ↑
服务层 Service（引用 schemas 作为返回类型）
         ↑
语义层 Semantic Layer（口径定义）
```

## 3. OpenAPI 集成流程

### 完整链路

| Step | 操作 | 说明 |
|------|------|------|
| ① | 定义 Pydantic 模型 | 在 `schemas.py` 中定义或修改 |
| ② | FastAPI 路由引用 | main.py 中使用 `response_model=...` |
| ③ | 生成 OpenAPI JSON | FastAPI 自动生成至 `/openapi.json` |
| ④ | 前端执行 `npm run gen:types` | 触发 openapi-typescript |
| ⑤ | 解析 OpenAPI | openapi-typescript 读取 JSON |
| ⑥ | 生成类型文件 | 输出 `src/api/types.ts`（1786行）|
| ⑦ | 前端获得类型推断 | import 后自动推断完整类型 |

### 生成命令

```bash
cd frontend-vue3
API_URL=http://localhost:8000/openapi.json npm run gen:types
```

### 生成结果

生成的 `src/api/types.ts` 包含：
- 所有 API 端点的 TypeScript 类型
- `paths` 联合类型：所有接口路径 + HTTP 方法
- 各 Response 模型的完整类型定义

## 4. main.py 集成规范

### ✅ 正确示例

```python
# main.py
from fastapi import APIRouter
from backend.contracts.schemas import OverviewMetrics, MetricTrendRequest

router = APIRouter()

@router.get("/api/v1/metrics/overview", response_model=OverviewMetrics)
async def get_overview(params: MetricTrendRequest) -> OverviewMetrics:
    ...
```

### ❌ 禁止内联

```python
# ❌ 禁止：main.py 中内联 Pydantic 模型
@router.get("/overview")
async def get_overview():
    return {
        "amount": float,   # 内联模型，禁止 ❌
        ...
    }
```

## 5. 前端类型使用规范

### 类型导入

```typescript
import type { paths } from '@/api/types'

// 自动获得完整类型推断
type OverviewResponse = paths['/api/v1/metrics/overview']['get']['responses']['200']['content']['application/json']['data']
```

### useApi Composable

`src/api/useApi.ts` 是基于 TanStack Query Vue 的封装，接收生成类型：

```typescript
import { useApi } from '@/api/useApi'
import type { paths } from '@/api/types'

const { data } = useApi('/api/v1/metrics/overview', {
  query: computed(() => ({
    start_date: '2026-01-01',
    end_date: '2026-01-31',
    metric_type: 'GMV'
  }))
})

// data.value 类型自动推断为 OverviewMetrics
```

## 6. 变更管理流程

| 步骤 | 操作 | 验证 |
|------|------|------|
| ① | 修改 `backend/contracts/schemas.py` | — |
| ② | 重启后端（确保 `/openapi.json` 更新）| 访问 `localhost:8000/openapi.json` |
| ③ | 前端执行 `npm run gen:types` | 检查 `src/api/types.ts` 行数变化 |
| ④ | 检查 types.ts 是否正确更新 | 确认新字段已出现 |
| ⑤ | 前端 `vue-tsc --noEmit` 验证 | 0 错误 |
| ⑥ | 提交两份文件 | `schemas.py` + `types.ts` |
