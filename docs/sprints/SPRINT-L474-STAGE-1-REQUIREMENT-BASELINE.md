# Sprint L4.74 Stage 1 需求与 DuckDB 性能基线

## 目标

锁定 PostgreSQL 16 POC 的 10 个业务查询场景，并保留 DuckDB 122GB 当前表现作为对照基线。

## 10 个场景

| # | 场景 | 当前风险 | 目标 |
|---:|---|---|---|
| 1 | 老客概览 30 天 | 高并发排队 | P95 < 5s |
| 2 | 复购周期 | 大 CTE | P95 < 5s |
| 3 | Cohort 留存 | 月度矩阵 | P95 < 5s |
| 4 | 价值分层 | 全量用户聚合 | P95 < 5s |
| 5 | 价值流转 | 三周期对比 | P95 < 5s |
| 6 | RFM 8 象限 | 122GB 全表聚合 | P95 < 5s |
| 7 | RFM 品类下钻 | 多 CTE + 品类 | P95 < 8s |
| 8 | 新客转化 | 首购/复购 join | P95 < 5s |
| 9 | 大促日历 | 多窗口对比 | P95 < 5s |
| 10 | 渠道健康评分 | 多渠道循环 | P95 < 8s |

## 当前基线

| 指标 | 当前值 |
|---|---|
| DuckDB 文件 | 122GB |
| PC2 并发用户 | 10 人 |
| 触发条件 b | 查询 P95 > 30s 持续 |
| 触发条件 c | 5+ 分析师并发取数 |
| 短期缓解 | FQ_READ_POOL_SIZE=10 + 预计算 |

## 采集方式

1. 对 10 个场景各跑 10 次，记录 P50/P95/P99。
2. 并发 1/5/10 三档压测。
3. 记录成功率、503 次数、fallback 次数。
4. PostgreSQL 16 单节点和 Citus cluster 用同一场景重跑。
