## 一、项目概览

| 维度 | 内容 |
|---|---|
| **项目名称** | 芙清 CRM 客户分析系统 (Fuqing CRM Analytics) |

5. **人群看板** — 渠道概览/地理/流失/RFM + 会员板块

## 四、架构核心原则

1. **语义层唯一真实数据源** — 口径只定义一次，禁止 Service 硬编码
2. **双保险过滤** — `is_refund=FALSE AND order_status!='交易关闭'`
3. **契约层外置** — Pydantic 模型统一从 `contracts/schemas.py` 导入
4. **前端只做展示** — 禁止前端计算业务指标
5. **连接零泄漏** — DuckDB 连接必须 try/finally 关闭
6. **API Key 强制检查** — 模块级检查，环境变量读取
7. **SQL 参数化** — DuckDB 用 `?` 参数，禁止字符串拼接

## 五、关键经验教训（历史 Bug 汇总）

| 严重度 | 问题 | 根因 | 修复 |
|---|---|---|---|
| P0 | `LIKE '%成功%'` 误杀 49k 订单 | 字符串匹配过于严格 | 双保险过滤 |
| P0 | API Key 硬编码 | 模块级无强制检查 | 环境变量 + 启动检查 |
| P1 | 新老客 cutoff 倒置 | 起始日计算错误 | T1-1 天逻辑 |
| P1 | RFM 预计算 vs 实时不一致 | F 基于不同时间窗口 | 统一全历史口径 |
| P1 | 占比 YOY 偏差 100 倍 | 小数 vs 百分比传入 | 量纲统一 |
| P1 | SQL 注入风险 | 字符串拼接 SQL | `?` 参数化 |
| P2 | CORS `["*"]` 开放 | 开发配置未收紧 | 环境变量白名单 |
