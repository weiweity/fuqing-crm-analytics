```python
# scripts/etl/preload_rfm.py:605-650 (run_auto_preload)
def run_auto_preload(today: date = None) -> List[Tuple[str, int]]:
    hot_dates = get_hot_dates(today)
    lookbacks = [30, 90, 180]
    metrics = ["GMV", "GSV"]
    channels = ["全店"] + ACTIVE_UI_CHANNELS  # 10 channels
    conn = duckdb.connect(str(DUCKDB_PATH), config={"memory_limit": DUCKDB_MEMORY_LIMIT})
    try:
        total_combos = len(lookbacks) * len(metrics) * len(channels)  # 60 combos
        for i, d in enumerate(hot_dates, 1):
            try:
                rows = preload_date_batch(conn, d, lookbacks, metrics, channels)  # ← 1 SQL/date
                results.append((d.isoformat(), rows))
                print(f"[{i}/{len(hot_dates)}] {d} => {rows:,} 行 ({total_combos} 组合/date, 1 SQL)")
            ...
```

**核心改动**: 1 SQL `preload_date_batch()` 替代 720 次串行 `preload_date()` 循环 (12x 减少)。代码层 ✅ 验证。

## 3. W1 GROUPING SETS 3 次跑批实测

### 3.1 跑批方法

**为什么用 wrapper 而非 --update**: `--update` 路径 Step 1 (run_full_etl) 撞到源数据 `(order_id, sub_order_id)` 重复 constraint error (见 §6), 无法跑完整 pipeline. 但痛点 1 根因 = W1 GROUPING SETS 性能, 不含 Step 1. 故用 wrapper 直接调 `run_auto_preload()` 验证 W1 单步 3 次.

**wrapper (inline python -c with PYTHONUNBUFFERED=1)**:

```python
def run_w1_once(idx):
    from scripts.etl.preload_rfm import run_auto_preload
    from scripts.etl._timer import PerfTimer, save_baseline, _RECORDS
    reset_records()  # 清空 stale per_step
    run_id = f'w1v2-{idx}/3'
    with PerfTimer('w1_run_auto_preload', run_id=run_id, idx=idx):
        results = run_auto_preload()
    payload = save_baseline(run_id=run_id, host='mac')
```

## 6. 已知 issue: --update Step 1 constraint error (独立)

### 6.1 错误

```
_duckdb.ConstraintException: Constraint Error:
  Duplicate key "order_id: 5119132537346076432, sub_order_id: 5119132537346076432"
  violates unique constraint.
```

文件: `scripts/etl/load.py:543-549` (INSERT INTO orders via ROW_NUMBER() PARTITION)

### 6.2 触发条件

- 源数据 (Parquet 缓存 `data/processed/taoke_file_cache.json` 63MB) 包含 (order_id, sub_order_id) 重复行
- 重复特征: order_id == sub_order_id (等同)
- dedup CTE `WHERE t._rn = 1` 应该去重, 但若 PARTITION BY (order_id, sub_order_id) 内多个 _rn=1 (排序键 pay_time DESC 相同), 仍会重复 INSERT

### 6.3 修复建议 (后续 sprint, 不在 P0-1)

1. **数据侧**: 排查 taoke_file_cache.json 中 order_id == sub_order_id 的源数据, 报告上游数据提供方
2. **ETL 侧**: 改 dedup 逻辑, 用 `DISTINCT ON (order_id, sub_order_id)` 或 `QUALIFY ROW_NUMBER() OVER (...) = 1` 提前过滤
3. **测试侧**: 加 fixture `(order_id, sub_order_id)` 重复场景, 防回归

### 6.4 P0-1 范围说明

痛点 1 = W1 GROUPING SETS 性能. constraint error 是数据 quality issue, 跟 W1 GROUPING SETS 改造无关. 故 P0-1 改用 wrapper 单独验 W1, 不在 worktree 改 dedup 逻辑 (会触发 12 步流程).
