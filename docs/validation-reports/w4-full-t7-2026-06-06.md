### 3.3 dbt-style snapshot version 续号验证

**调用次数** (test_a + test_b + test_c 各跑一次) **= 3 次**

每次调用内部:
- `incremental_load(T-1)` → version = `_next_version(T-1) = MAX(T-1) + 1`
- `merge_replace(T-1)` → version = `_next_version(T-1) = MAX(T-1) + 1` (再次 +1)
- `merge_replace(T-i)` (i=2..7) → version = `_next_version(T-i) = MAX(T-i) + 1`

**T-1 version 演化**:
| 调用 | 增量 load version | merge T-1 version | T-1 累计 version |
|---|---|---|---|
| 1 (test_a) | 1 | 2 | [1, 2] |
| 2 (test_b) | 3 | 4 | [1, 2, 3, 4] |
| 3 (test_c) | 5 | 6 | [1, 2, 3, 4, 5, 6] |

实测: T-1 max version = 6 ✓ — **version 续号正确, 6 个版本都存在**

**T-2..T-7 version 演化**:
| 调用 | merge T-i version | T-i 累计 version |
|---|---|---|
| 1 | 1 | [1] |
| 2 | 2 | [1, 2] |
| 3 | 3 | [1, 2, 3] |

实测: T-2..T-7 max version = 3 ✓ — **version 续号正确, 3 个版本都存在**
