## 2. filters.py — 过滤条件

### 2.1 核心过滤函数

| 函数 | SQL 片段 | 用途 |
|------|---------|------|
| `OrderFilters.not_refund()` | `order_status != '交易关闭' AND is_refund = FALSE` | 退款双重保险 |
| `OrderFilters.not_goujinjin()` | `is_goujinjin = FALSE` | 剔除购物金 |
| `OrderFilters.valid_order()` | `is_goujinjin = FALSE AND order_status != '交易关闭' AND is_refund = FALSE` | **GSV 口径** |
| `OrderFilters.gmv_base()` | `is_goujinjin = FALSE AND order_status != '交易关闭'` | **GMV 口径（含退款）** |
| `OrderFilters.order_status_ok()` | `order_status != '交易关闭' AND is_refund = FALSE` | ⚠️ **已废弃**，请使用 `valid_order()` |

> ⚠️ `order_status_ok()`（`LIKE '%成功%'`）**已废弃**，于 2026-04-17 删除。所有有效订单判定必须使用 `valid_order()`（双保险）。

### 2.2 FilterBuilder — 动态条件构造器

```python
from backend.semantic.filters import FilterBuilder, MetricType, OrderFilters

# 构造 GSV 口径 + 渠道筛选 + 时间范围
fb = FilterBuilder()
fb.with_metric_type(MetricType.GSV)          # 自动选 valid_order()
fb.with_time_range("2026-01-01", "2026-01-31")
fb.with_channels(["U先派样", "百补派样"])       # UI名自动映射为 DB名
fb.with_exclude_channels(["U先派样", "百补派样", "赠品&0.01", "其他"])  # 低价剔除
fb.with_member_only(True)

sql, params = fb.build()
# sql => "pay_time >= ? AND pay_time <= ? AND is_goujinjin = FALSE AND
#         order_status != '交易关闭' AND is_refund = FALSE AND
#         channel IN (?, ?) AND is_member = TRUE"
```

### 2.3 AmountExprBuilder — 金额表达式

```python
from backend.semantic.filters import AmountExprBuilder

# 在同一 SQL 中同时计算 GSV 和 GMV
gsv_expr = AmountExprBuilder.gsv()       # CASE WHEN (...) THEN actual_amount ELSE 0
gmv_expr = AmountExprBuilder.gmv()      # actual_amount（直接使用列）
sum_gsv  = AmountExprBuilder.sum_gsv()  # SUM(CASE WHEN (...) THEN actual_amount ELSE 0)
```

## 3. metrics.py — 指标注册表

### 3.1 核心指标一览

| 指标 key | 中文名 | SQL 表达式 | 口径说明 |
|---------|--------|-----------|---------|
| `gmv` | GMV | `SUM(CASE WHEN is_goujinjin=FALSE THEN actual_amount ELSE 0 END)` | 含退款 |
| `gsv` | GSV | `SUM(CASE WHEN (is_goujinjin=FALSE AND order_status!='交易关闭' AND is_refund=FALSE) THEN actual_amount ELSE 0 END)` | 剔除退款 |
| `member_gsv` | 会员GSV | 同上 + `is_member=TRUE` | 会员有效销售 |
| `new_gsv` | 新客GSV | `SUM(CASE WHEN is_new=1 THEN amount ELSE 0 END)` | 窗口首购用户 |
| `old_gsv` | 老客GSV | `SUM(CASE WHEN is_old=1 THEN amount ELSE 0 END)` | 历史有购买 |
| `order_count` | 订单数 | `COUNT(DISTINCT order_id)` | 去重 |
| `gsv_order_count` | 有效订单数 | `COUNT(DISTINCT CASE WHEN valid_order THEN order_id END)` | GSV口径 |
| `total_users` | 购买人数 | `COUNT(DISTINCT user_id)` | 去重用户 |
| `gsv_users` | 有效购买人数 | `COUNT(DISTINCT user_id)` | GSV口径 |
| `new_users` | 新客人数 | `COUNT(DISTINCT CASE WHEN is_new=1 THEN user_id END)` | 窗口首购 |
| `old_users` | 老客人数 | `COUNT(DISTINCT CASE WHEN is_old=1 THEN user_id END)` | 窗口有购且历史有购 |
| `avg_order_value` | 客单价 | `AVG(actual_amount)` | GMV/订单数 |
| `aus` | 人均消费 | `SUM(actual_amount) / NULLIF(COUNT(DISTINCT user_id), 0)` | GSV/有效人数 |
| `member_gsv_ratio` | 会员GSV占比 | `member_gsv / gsv` | 百分比 |
| `old_gsv_ratio` | 老客GSV占比 | `old_gsv / gsv` | 百分比 |

## 4. calculations.py — 统一计算规则（v4.0 新增）

> **2026-04-20 完成统一**：所有 MoM/YoY 计算必须使用此模块，禁止在 Service 中内联。

### 4.1 函数一览

| 函数 | 用途 | 公式 |
|------|------|------|
| `yoy_absolute()` | 绝对值 YOY | `(cur - comp) / comp` → 小数（如 0.25 = 25%） |
| `yoy_ratio()` | 占比 YOY（百分点差） | `cur - comp` → 百分点（如 0.05 = 5pp） |
| `mom_absolute()` | 绝对值 MoM | `(cur - prev) / prev` → 小数 |
| `mom_ratio()` | 占比 MoM（百分点差） | `cur - prev` → 百分点 |
| `safe_ratio()` | 安全除法 | `a / b 或 default`，避免 ZeroDivisionError |

### 4.2 使用示例

```python
from backend.semantic.calculations import yoy_absolute, yoy_ratio, mom_absolute, mom_ratio

# ✅ 正确：使用统一函数
gsv_yoy = yoy_absolute(current_gsv, last_year_gsv)       # 金额 YOY（返回小数）
old_ratio_yoy = yoy_ratio(current_old_ratio, last_year_old_ratio)  # 占比 YOY（返回百分点）
gsv_mom = mom_absolute(current_gsv, prev_month_gsv)     # 金额 MoM
ratio_mom = mom_ratio(current_ratio, prev_ratio)           # 占比 MoM

# ❌ 错误：Service 内联计算
gsv_yoy = (current_gsv - last_year_gsv) / last_year_gsv * 100  # 禁止！
```

### 4.3 前端展示约定

- **绝对值 YOY**（金额/人数/订单数）：后端返回小数，前端展示 `%`
- **占比 YOY**（比率/百分比）：后端返回百分点差，前端展示 `pp`

### 5.2 RFM 阈值（v4.0）

```python
RFM_THRESHOLDS = {
    "r": [14, 30, 60, 90],    # 分值 5/4/3/2/1（≤14天=5分，≤90天=2分）
    "f": [1, 2, 3, 5],        # 分值 1/2/3/4/5（≥6次=5分）
    "m": [100, 300, 500, 1000]  # 分值 1/2/3/4/5
}
```

### 5.3 使用方式

```python
from backend.semantic.segments import SegmentRegistry, get_registry, RFM_THRESHOLDS

registry = get_registry()

# 生成 R/F/M 评分 SQL
r_score_sql = registry.build_r_score_sql()
f_score_sql = registry.build_f_score_sql()
m_score_sql = registry.build_m_score_sql()

# 生成象限 CASE WHEN
segment_sql = registry.build_segment_case_when_sql()

# 获取象限名称/颜色
name = registry.get_name_cn(segment_id)
color = registry.get_color(segment_id)
```

## 6. channels.py — 9 层渠道漏斗（v4.0）

### 6.1 渠道映射（DB ↔ UI 统一）

> **v4.0 重大变更**：数据库存储统一为 `U先派样`（大写U），与 UI 一致。

| DB 实际名 | UI 展示名 | 说明 |
|---------|---------|------|
| `U先派样` | U先派样 | **统一大写U**（v4.0 前：u先派样） |
| `百补派样` | 百补派样 | 一致 |
| `赠品&0.01渠道` | 赠品&0.01 | 多"渠道"后缀 |
| `达播` | 达播 | 一致 |
| `微博` | 微博 | 一致 |
| `直播` | 直播 | 一致 |
| `淘客` | 淘客 | 一致 |
| `货架` | 货架 | 一致 |
| `其他` | 其他 | 一致 |

**映射位置（唯一数据源）**：
- `backend/semantic/channels.py` → `DB_TO_UI` / `UI_TO_DB`
- `backend/semantic/filters.py` → `_CHANNEL_GROUP_MAP`
- `backend/services/metrics_service.py` → `_expand_channel()`

### 6.2 漏斗层级

| 层级 | 名称 | 判定规则 | 保护 |
|------|------|---------|------|
| P1 | U先派样 | spu_type 含"小样-U先" 或 product_title 含"U先" | — |
| P2 | 百补派样 | spu_type 含"小样-百亿补贴" 或 product_title 含"by" | — |
| P3 | 赠品&0.01 | spu_type 含"小样"（排除P1/P2）或 product_title 含"赠品" 或 amount < 4 | — |
| P4 | 达播 | keyword_rules / id_rules | ✅ 仅对 `channel='其他'` 生效 |
| P5 | 微博 | keyword_rules / id_rules | ✅ 仅对 `channel='其他'` 生效 |
| P6 | 直播 | order_id 在直播 CSV 父订单号集合中 | ✅ 仅对 `channel='其他'` 生效 |
| P7 | 淘客 | order_id 在淘客数据库中 | ✅ 仅对 `channel='其他'` 生效 |
| P8 | 货架 | P1-P7 未命中且 spu_type 含"正装" | — |
| P9 | 其他 | P1-P8 未命中 | — |

> ⚠️ P4/P5/P6/P7 都需要通过 `channel == '其他'` 保护 P1-P3，v4.0 修复了此前 P4 直接覆盖的 bug。

## 7. time.py — 时间周期构造器

| 方法 | 说明 | 示例 |
|------|------|------|
| `this_month()` | 当月至今 | 2026-04-01 ~ 2026-04-20 |
| `last_month()` | 上月同期 | 2026-03-01 ~ 2026-03-20 |
| `same_period_last_year()` | 去年同期 | 2025-04-01 ~ 2025-04-20 |
| `wtd()` | **当周至今（默认）** | 2026-04-13 ~ 2026-04-20 |
| `prev_wtd()` | 上周同期 | 2026-04-06 ~ 2026-04-12 |
| `prev2_wtd()` | 上上周同期（v2.0 修复） | 2026-03-30 ~ 2026-04-05 |

> ⚠️ `prev2_wtd` 之前存在日期倒置 Bug（v2.0 已修复）。
