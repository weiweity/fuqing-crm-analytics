## 一、知识调整：渠道方向

### 1.1 芙清渠道漏斗（10层，含购物金剔除）

```yaml
entity: 芙清渠道漏斗
type: 流程步骤
category: 渠道管理
platform: 芙清CRM
source: backend/semantic/channels.py
version: 2026-04-27

有效渠道（ACTIVE_CHANNELS）：P1-P6、P8-P9，共8个（不含P7购物金）

层级说明（互斥匹配，按优先级从高到低）：

P1. U先派样
    - 识别：spu_type含'小样-U先' 或 product_title含'U先'/'会员尝鲜'
    - 性质：派样获客渠道

P2. 百补派样
    - 识别：spu_type含'小样-百亿补贴' 或 product_title含'by'
    - 性质：派样获客渠道

P3. 赠品&0.01
    - 识别：spu_type含'小样'(排除U先/百补) 或 product_title含'赠品'
    - 性质：派样获客渠道（0.01锁权+赠品派发）

P4. 达播
    - 识别：keyword_rules/id_rules匹配达播来源
    - 性质：达人直播获客渠道（与微博同层，独立渠道）
    - 运营特征：
      * 订单集中在大促/专场活动爆发
      * 退货率通常高于自然流量
      * 新客占比高，需CRM二次触达召回

P4. 微博
    - 识别：keyword_rules/id_rules匹配微博来源
    - 性质：社交渠道（与达播同层，独立渠道）
    - 注意：与达播是独立渠道，不是同一个

P5. 直播（店播）
    - 识别：order_id匹配直播CSV父订单号（仅对未匹配订单生效）
    - 性质：店铺自播获客渠道
    - 运营特征：
      * 日常稳定出单
      * 会员渗透率较高
      * 需配合会员专属直播权益

P6. 淘客
    - 识别：P6订单号匹配 或 P6-2标题含T1/T2/T4/TK（仅对未匹配订单生效）
    - 性质：按成交付费的投放渠道

P6-2. 淘客关键词（T1/T2/T4/TK）
    - T1：淘客专项引流款
    - T2：淘客进阶款
    - T4：淘客高端款
    - TK：淘客主推款
    - 识别：product_title含T1/T2/T4/TK后缀
    - 注意：仅对channel='其他'的未匹配订单生效

P7. 购物金
    - 识别：product_title含'购物金'
    - 性质：蓄水工具，ETL标记后剔除
    - 重要：P7购物金不计入有效渠道（ACTIVE_CHANNELS已剔除）
    - GSV口径：含购物金，含退款；GSV口径不含购物金

P8. 货架
    - 识别：P1-P7未命中 且 spu_type含'正装'
    - 性质：自然流量/搜索/推荐获客

P9. 其他
    - 识别：P1-P8未命中
    - 性质：兜底渠道
```

### 1.2 达播CRM联动策略

```yaml
entity: 达播CRM联动
type: 策略矩阵
category: 渠道策略
platform: 芙清CRM
source: 运营经验

达播前（活动前7天）：
- 人群准备：高价值老客/会员名单提供给达人运营
- 预约机制：老客会员专属预约权益，优先抢购权

- 实时监控：达播订单占比、达播新客占比
- 实时触达：直播间互动用户推送专属客服消息
- 达播专属权益：直播间锁权用户打标

达播后（后7天）：
- 新客召回：达播未购用户短信召回
- 退货预警：达播高客单价订单跟踪
- 会员转化：达播新客引导入会
```

## 二、知识调整：拆解方向

### 2.1 老客拆解：芙清版

```yaml
entity: 芙清老客RFM分析
type: 流程步骤
category: 数据分析
platform: 芙清CRM
synonyms: [芙清老客回购预估, 芙清RFM拆解]
related: [RFM, 回购率, GMV预估, 渠道漏斗]
source: MEMORY.md + 运营实践
version: 2026-04-27

芙清RFM阈值（非均匀分布）：
- R: [30, 90, 180, 365] 天
- F: [1, 2, 3, 4] 次
- M: [100, 300, 500, 1000] 元

芙清特色：
- 非线性评分V2：非均匀分数分布，更精准识别高价值客户
- 预计算缓存：DuckDB预计算表 + JSON文件缓存
- 跨Tab联动：RFM评分 → 渠道归因 → 品类偏好 → 流失预警

新老客基准：
- cutoff = T1 - 1天
- lookback_days：RFM窗口参数

芙清老客拆解公式（与天猫通用一致，但参数不同）：
- 公式：老客GMV = 老客人数 × 回购率 × 客单价
- 超头处理：达播(P4)已从漏斗中独立，店铺自然老客不含达播
- 达播退货率监控：达播订单退货率通常高于自然流量，需单独跟踪

分析Tab结构（芙清仪表盘）：
- Tab1：概览（GMV/人数/客单价/转化率）
- Tab2：象限矩阵（R×F×M三维交叉）
- Tab3：价值分层（高/中/低/流失）
- Tab4：品类流转（各RFM层级的品类偏好）
- Tab5：流失预警（高R低F预警）
- Tab6：购物篮分析（RFM×品类交叉）
```

### 2.2 新客拆解：芙清版

```yaml
entity: 芙清新客拆解
type: 流程步骤
category: 数据分析
platform: 芙清CRM
synonyms: [芙清新客预估, 芙清拉新拆解]
related: [新客, 渠道漏斗, 入会率]
source: 运营实践
version: 2026-04-27

新客定义：
- 天猫通用口径：店铺维度未购
- 芙清口径：同天猫（cutoff=T1-1天）

新客来源拆解（芙清版，按10层渠道漏斗）：
- 达播新客：P4达播渠道的新客订单
- 微博新客：P4微博渠道的新客订单
- 店播新客：P5直播渠道的新客订单
- 淘客新客：P6/P6-2淘客渠道的新客订单
- 货架新客：P8货架渠道的新客订单
- 其他新客：P9其他渠道的新客订单

芙清新客预估公式：
- 各渠道新客 = 各渠道订单人数 × (1 - 老客占比)
- 全店新客 = 各渠道新客之和

入会率跟踪：
- 各渠道招募量 ÷ 各渠道UV = 入会率
- 重点监控：P1 U先派样 → 入会转化链路
```

### 2.3 会员拆解：芙清版

```yaml
entity: 芙清会员指标体系
type: 指标公式
category: 数据分析
platform: 芙清CRM
synonyms: [芙清会员分析, 芙清会员指标]
related: [会员, RFM, 渠道漏斗]
source: 人群看板 + 运营实践
version: 2026-04-27

芙清会员核心指标（人群看板Tab）：
- 会员占比：会员GMV / 全店GMV
- 会员YoY：同比会员指标变化
- 交叉指标：会员 × 渠道 / 品类 / 地区 / RFM

芙清会员×渠道交叉分析：
- 各渠道的会员渗透率 = 该渠道会员GMV / 该渠道总GMV
- 会员渠道偏好：高会员占比渠道 = 高忠诚度渠道
- 重点监控：P5直播(店播)会员渗透率通常高于其他渠道

会员×RFM交叉：
- 会员高价值：已购-会员象限 × 高M值
- 会员唤醒：已购-会员象限 × 高R值
- 会员流失预警：已购-会员象限 × 高R值 + 低F值
```

### 2.4 芙清活动拆解：五步法

```yaml
entity: 芙清活动规划拆解
type: 流程步骤
category: 核心工作流
platform: 芙清CRM
source: 芙清CRM仪表盘 + 运营实践
version: 2026-04-27

Step1: 拍目标
- 输入：芙清历史同期数据（生意参谋）、市场洞察大盘趋势、竞品监控
- 输出：全店GMV目标、新老客占比目标、会员渗透目标
- 芙清工具：CEM + 生意参谋 + 芙清Dashboard

Step2: 做预估
- 老客预估：芙清RFM分层 × 各层回购率 × 各层客单价
  * 使用芙清预计算缓存（DuckDB + JSON）
  * 达播订单独立统计，不计入店铺自然老客
- 新客预估：各渠道新客之和（按10层漏斗拆分）
  * 重点：达播新客 + 淘客新客 + 货架新客
- 会员预估：各渠道招募量 × 入会率
  * 重点：P1 U先派样 → P3赠品&0.01 的入会转化链路

Step3: 找gap
- gap = 目标值 - 预估值
- 芙清分析维度：
  * 人群维度：新客gap / 老客gap / 会员gap
  * 渠道维度：各渠道GMV缺口
  * 品类维度：各品类GMV缺口
  * 地理维度：各地区GMV缺口

Step4: 争机制
- 补gap策略：
  * 老客gap：加深offer、增加触达轮次（芙清CRM触达）
  * 新客gap：扩大派样量（U先/百补/赠品）、提升入会率
  * 会员gap：提升会员权益、会员专属机制

Step5: 落机制
- 物资清单：优惠券/赠品/页面权益
- 触达排期：CRM触达计划（短信/群消息）
- 达播协调：与达人提前确认排期、机制、预热素材
```

## 四、天猫通用 vs 芙清：拆解方向对比

| 维度 | 天猫通用（知识萃取原文） | 芙清实际 | 调整方式 |
|------|------------------------|---------|---------|
| 老客拆解框架 | RFM四步法，人工表格 | 仪表盘自动计算，6Tab联动 | 替换为芙清版 |
| RFM阈值 | 天猫后台标准分段 | 自定义R[30/90/180/365] F[1/2/3/4] M[100/300/500/1000] | 替换 |
| 回购预测 | 历史回购率×人数×客单价 | 非线性评分V2 + 预计算缓存 | 增强 |
| 超头处理 | 拆两套（超头老客+店铺老客） | 达播(P4)已独立，店铺自然老客不含达播 | 替换 |
| 新客拆解 | 超头新客+店铺新客 | 达播+微博+店播+淘客+货架+其他（10层渠道） | 扩展 |
| 会员分析 | 黄金四象限，概念性 | 人群看板会员Tab，会员×渠道/品类/地区交叉 | 增强 |
| 活动拆解 | 五步法（拍目标→做预估→找gap→争机制→落机制） | 同框架，工具替换为芙清Dashboard | 本地化 |
