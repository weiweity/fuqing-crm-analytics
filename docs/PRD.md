# 芙清 CRM 客户分析系统 - 产品需求文档 (PRD)

**版本**: v1.0  
**日期**: 2026-03-27  
**作者**: 运营团队  
**状态**: 已确认

---

## 1. 产品概述

### 1.1 产品名称
芙清 CRM 客户分析系统 (Fuqing CRM Analytics)

### 1.2 产品定位
内部运营中台工具，为芙清电商运营团队提供数据驱动的客户洞察能力，支撑日常运营决策和月度复盘。

### 1.3 核心价值主张
- **每日 9 点自动推送**：运营团队每天上班即获得最新客户洞察
- **智能问数**：通过自然语言查询，快速获取数据答案
- **一键导出复盘**：自动生成 PPT/飞书文档，支撑月度复盘会议

### 1.4 目标用户

| 用户角色 | 使用频率 | 核心场景 | 成功标准 |
|---------|---------|---------|---------|
| 运营专员（你） | 每天 | 查看数据、发现问题、生成洞察 | 5 分钟内获得可行动的洞察 |
| 运营经理 | 每天 | 掌握团队整体情况、分配资源 | 3 分钟了解全局 |
| 老板/创始人 | 每周/每月 | 听汇报、参与复盘 | 10 分钟理解业务现状 |

---

## 2. 核心问题与痛点

### 2.1 现状痛点
1. **无数据驱动**：产品迭代凭感觉，不知道为什么爆、为什么好
2. **Excel 太慢**：做完分析洞察已过时，无法支撑实时决策
3. **文件太大**：2GB 数据无法追溯历史，做不了长期分析
4. **机会唯一**：全公司只有你能做，是升职带团队的关键项目

### 2.2 成功指标
- **效率提升**：数据获取时间从 2 小时降至 5 分钟
- **洞察时效**：从 T+7 提升至 T+1（次日 9 点）
- **决策支撑**：每周至少产生 1 个可执行的运营建议

## 3. 功能需求

### 3.1 Week 1: 核心指标看板

#### 3.1.1 数据清洗与标准化
- **输入**: 25-26 年店铺数据库 + 会员数据库（约 50 个 xlsx 文件）
- **输出**: 清洗后的 DuckDB 数据库
- **清洗规则**:
  - 统一时间格式（下单时间、付款时间）
  - 标准化商品标题（提取 SKU、规格、品类）
  - 用户 ID 去重与关联
  - 金额字段统一（元为单位，处理退款）

#### 3.1.2 核心指标模块

| 模块 | 指标 | 计算逻辑 | 展示方式 |
|------|------|---------|---------|
| **GMV 概览** | 总 GMV | 实付金额汇总 | 卡片 + 趋势图 |
| | 订单数 | 去重订单编号计数 | 卡片 + 趋势图 |
| | 客单价 | GMV / 订单数 | 卡片 |
| **新老客分析** | 新客数 | 当月首次购买用户数 | 卡片 + 饼图 |
| | 老客数 | 当月复购用户数（2月1日前买过） | 卡片 + 饼图 |
| | 新客 GMV | 新客实付金额 | 卡片 |
| | 老客 GMV | 老客实付金额 | 卡片 |
| **会员分析** | 会员数 | 有会员标识的用户数 | 卡片 |
| | 会员 GMV | 会员实付金额 | 卡片 |
| | 会员占比 | 会员 GMV / 总 GMV | 百分比 |
| **同环比** | 同比 | 当月 vs 去年同期 | 增长率 + 箭头 |
| | 环比 | 当月 vs 上月 | 增长率 + 箭头 |

#### 3.1.3 筛选能力
- 时间范围：日/周/月/自定义
- 店铺：芙清旗舰店（默认，后续可扩展）
- 产品维度：全部/单品类/单 SKU

### 3.2 Week 2: RFM 模型与 618 决策支撑

#### 3.2.1 R 区间分析（最近购买时间）
| 区间 | 定义 | 运营策略 |
|------|------|---------|
| R0-30 | 30 天内购买 | 重点维护，推送新品 |
| R31-60 | 31-60 天 | 温和唤醒，优惠券刺激 |
| R61-90 | 61-90 天 | 强力召回，大促触达 |
| R90+ | 90 天以上 | 沉睡客户，低成本维护 |

#### 3.2.2 F 区间分析（购买频率）
| 区间 | 定义 | 运营策略 |
|------|------|---------|
| F1 | 购买 1 次 | 新客转化，引导复购 |
| F2-3 | 购买 2-3 次 | 潜力客户，会员引导 |
| F4-6 | 购买 4-6 次 | 忠诚客户，专属权益 |
| F7+ | 购买 7 次以上 | 超级用户，VIP 服务 |

#### 3.2.3 M 区间分析（购买金额）
| 区间 | 定义 | 运营策略 |
|------|------|---------|
| M0-100 | 100 元以下 | 低价值，批量运营 |
| M100-300 | 100-300 元 | 中价值，常规运营 |
| M300-500 | 300-500 元 | 高价值，重点维护 |
| M500+ | 500 元以上 | 超高价值，1v1 服务 |

#### 3.2.4 RFM 综合模型
- 8 象限客户分层（高价值活跃、高价值沉睡、低价值活跃等）
- 618 人群包推荐：该唤醒谁、该维护谁、该放弃谁

---

### 3.3 Week 3: 人群流转与资产分析

#### 3.3.1 人群流转矩阵
- 展示上月 → 本月的人群状态变化
- 例如：新客 → 老客、活跃 → 沉睡、沉睡 → 回流

#### 3.3.2 人群资产看板
| 指标 | 定义 | 价值 |
|------|------|------|
| 总客户资产 | 历史累计客户数 | 品牌积累 |
| 活跃资产 | 近 90 天有购买 | 核心运营对象 |
| 沉睡资产 | 90 天未购买 | 召回潜力 |
| 流失资产 | 180 天未购买 | 低成本维护 |

#### 3.3.3 单品资产分析
- 每个 SKU 的种草 → 转化 → 复购漏斗
- 识别流转环节问题（哪个环节流失最多）

---

### 3.4 Week 4: 人群画像与趋势分析

#### 3.4.1 人群画像维度
- **地域分布**：省份、城市 TOP10
- **购买时段**：小时分布、星期分布
- **品类偏好**：各品类购买占比
- **价格敏感度**：折扣购买占比

#### 3.4.2 月度变化追踪
- 每个月人群画像的对比
- 识别人群质量变化趋势

---

### 3.5 Week 5: 缺口追踪与预测

#### 3.5.1 缺口诊断
- 业绩缺口 = 目标 GMV - 实际 GMV
- 缺口归因：新客缺口 vs 老客缺口
- 可执行建议：需要多少新客？需要多少复购？

#### 3.5.2 趋势预测
- 基于历史数据预测下月 GMV
- 预测置信区间
- 提前预警风险

---

## 4. 非功能需求

### 4.1 性能需求
- 数据查询响应时间 < 3 秒
- 页面加载时间 < 5 秒
- 支持 2GB 数据量秒级查询

### 4.2 可用性需求
- 每日 9 点自动刷新数据
- 支持飞书机器人推送
- 一键导出 PPT/Excel

### 4.3 安全需求
- 本地运行，不对外暴露
- 数据不离开本地电脑

## 5. 数据定义

### 5.1 老客定义（月维度滚动窗口）
```
以 2026-02-25 为例：
- 老客 = 在 2026-02-01 之前有过购买记录的用户
- 新客 = 在 2026-02-01 至 2026-02-25 首次购买的用户
```

### 5.2 会员定义
- 有会员标识的用户（根据数据中的会员字段判断）

### 5.3 GMV 定义
- GMV = 实付金额（已扣除退款）
