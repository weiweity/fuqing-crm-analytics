## 改动 1：navigator.webdriver 反检测注入

**文件**: `core/dmp_common.py`
**位置**: `BrowserManager.__enter__()`，persistent_context 创建之后

**内容**:
```python
self._browser.add_init_script("""
    Object.defineProperty(navigator, 'webdriver', {
        get: () => false,
        configurable: true
    });
""")
```

**作用**:
- 达摩盘/Taobao 前端 JS 会检查 `window.navigator.webdriver === true` 来判断是否自动化浏览器
- `--disable-blink-features=AutomationControlled` 参数只能骗过部分检测，`navigator.webdriver` 需要显式覆写
- `add_init_script` 在每个新 page 创建时自动执行，早于任何页面 JS，无法被检测为"注入"
- 性价比最高的反爬虫改动

**影响范围**: 所有通过 `BrowserManager` 创建的 page（三个模块全部覆盖）

## 改动 2：断点续传缓存 `_mark_completed` 写入

**文件**: `core/dmp_master.py`
**位置**: `run_items_module()` 函数内，两处

### 位置 A（第一轮写入成功后）
```python
if dmp_item_insight_scraper.append_to_csv(Config.ITEM_DATA_FILE, data):
    success_count += 1
    try:
        dmp_item_insight_scraper._mark_completed(item_id, date_str_task)
    except Exception as e:
        log(f"⚠️ 写入完成缓存失败（非致命）: {e}")
    log(f"✓ 商品 {item_id} 日期 {date_str_task} 写入成功")
```

### 位置 B（重试轮写入成功后）
```python
if dmp_item_insight_scraper.append_to_csv(Config.ITEM_DATA_FILE, data):
    success_count += 1
    retry_success += 1
    retry_count += 1
    try:
        dmp_item_insight_scraper._mark_completed(item_id, date_str)
    except Exception as mc_err:
        log(f"⚠️ 写入完成缓存失败（非致命）: {mc_err}")
    log(f"✓ [重试成功] 商品 {item_id} 日期 {date_str}")
```

**背景**:
- `_mark_completed()` 在 `dmp_item_insight_scraper.py` 里早已实现，但从未被调用
- `_load_completed_items()` 在 `dmp_master.py` 里被调用来读取缓存做断点续传
- 缓存从未被写入，导致每次运行都从头抓所有数据（不致命，但效率低）
- 现在写入成功后会自动写入 `completed_items.json`，下次运行会自动跳过已完成的任务

**影响范围**: 仅单品洞察模块（Module 3）
